# Get-to-Know-You — builds the Context Vault (BOUNDED — terminates by design)

**What this is:** the short interview that gives every specialist real context about the user.
It fills the Context Vault (`memory/context/` + `memory/people/`). Run once at onboarding
(Stage 4 of the wizard); it deepens later through use — never through longer interviews now.

**The bar (Minimum Viable Profile — the ONLY requirement to finish):** four fields —
1. **Identity** — who they are, in a paragraph.
2. **The one big goal** — what they're building toward.
3. **How they like to work** — tone, cadence, how much autonomy.
4. **Key people** — the few people in their world ("none for now" is a valid answer).

**Hard mechanics (these guarantee it ends):**
- **Docs first.** *"Drop me anything that already exists — bio, CV, LinkedIn, goals note. I'll read
  it and only ask what's missing."* Never make anyone generate from a blank page; never ask what
  their material already answers. **Ask the date of any CV/bio before treating it as current**
  (stale facts written as truth violate truth-over-aspiration at intake; the date question is
  free — intake hygiene, not a gap question, so it never consumes budget). No docs? Fine — go
  straight to the questions.
- **Question budget: 5, total, hard — persisted.** Only for empty fields. Propose inferable defaults
  to confirm (timezone, tone) instead of asking cold. Count questions in STATE.md
  (`questions_asked_stage4`) so an interruption never resets the budget. Four fields filled OR five
  questions asked — whichever first — **STOP. Never ask a sixth.** Unfilled fields are written as
  "unknown — deepen later" and do not block anything.
- **Write each answer as received.** Answers go into `memory/context/profile.md`,
  `memory/context/preferences.md`, `memory/people/*.md` the moment they're given — never held only
  in chat (a closed app must lose nothing). When you stop, mirror FROM the files:
  *"Here's who I think you are — already written into your Vault, these files exist in your folder:
  … Tell me where I'm wrong."* Corrections = file edits, not reopened interviews.
- **No open-ended probing here.** Deep probe-and-challenge belongs to later working sessions. "No
  time pressure" does NOT mean "no stop condition" — the stop condition above is absolute.

**Depth later (the standing offer):** *"Whenever you want me to know you deeper, say 'deepen my
vault'."* That runs `VAULT-GUIDE.md` (the fuller vault: values under pressure, energy map, underused
leverage, your bar) — optional, come-back-later, also bounded per session.

**Instant proof → first task:** end with one specific observation that bridges to work:
*"Given you're after X and Y drains you, the first thing I'd take off your plate is Z — want me to?"*

**Rules of the Vault:** truth over aspiration — record what's true now, update as reality changes,
never flatter. Files are created progressively, never as blank scaffolding.
