# Context Vault — the schema (what the user's memory holds, and where)

_The Vault is what makes every specialist personal instead of generic. It lives in the USER'S
folder, is filled by the get-to-know-you interview (`get-to-know-you.md`), and deepens through the
learning loop and `VAULT-GUIDE.md`. Every mode reads it — knowing someone once makes the whole team
act like it's known them for years._

## The files (this layout is the single source of truth — it matches the boot order in CLAUDE.md)
| File | Holds | Created |
|---|---|---|
| `memory/context/profile.md` | identity (who they are, their days) · the one big goal / who they're becoming · reality & constraints · values and energy map as they emerge | at onboarding (the MVP), deepened forever |
| `memory/context/preferences.md` | how they like to work: tone, ping hours, updates, purchase rule, "your bar" · **the autonomy-grants ledger** (the ONE ledger) · tools granted/declined · standing commitments | at onboarding, from `preferences-template.md` |
| `memory/people/<name>.md` | family, close friends, key contacts — one file each | as people come up |
| `memory/context/VAULT-GUIDE.md` | the optional deepening areas (values under pressure · energy map · underused leverage · who they're becoming · their bar) | placed at setup; runs on "deepen my vault" |
| `memory/log/<date>.md` | session hand-offs ("wrap up"): what happened, decisions, open items | every wrap-up |

## Rules
1. **Create files progressively** — only as they fill. Never blank scaffolding on day one.
2. **Truth over aspiration** — record what's true now; update as reality changes; never flatter.
3. **Depth compounds** — onboarding captures enough to start (identity · goal · working style ·
   people); the learning loop and "deepen my vault" do the rest through real use.
4. **One person, one Vault, one layout** — deepening grows the files above; never invent a parallel
   set of files.

## For a cold session
To understand the person: read profile.md + preferences.md (+ people as needed). To build them:
run `get-to-know-you.md`. To deepen one area later: VAULT-GUIDE.md, one bounded area per session.
